# PKI

This directory contains all cryptographic material used in the system.

> This dir is not commited to the repository (see `.gitignore`).

All files inside are generated by:
`scripts/init_pki.sh`

## Purpose
This project uses `Public Key Infrastructure` where `Trusted Third Part` is used as `Certificate Authority`

### The PKI is responsible for:
- establishing a root of trust
- issuing X.509 certificates
- enabling authentication and secure key exchange

## Dir Structure
- `private`
  - contains private keys (TTP CA private key)
  - Shouldn't be commited to repository
- `trust`
  - contains public certificates (TTP CA public certificate)
- `csr`
  - generated during registration
  - contains Certificate Signing Request
- `issued`
  - contains certificates issued to client and server

## Security Model
- the TTP acts as a root of trust (CA).
- the CA key is generated locally and must remain secret.
- client and server generate their own private keys.
- certificates are issued by the TTP during registration.
